### 課題1: 括弧書きまで読み上げてしまう
`front` に `（ ）` の注釈や補足が入っているカードがあり、`speechSynthesis` がカッコの中身まで読んでしまう。
→ **対策：読み上げ前に括弧・注釈を除去する `stripForSpeech()` を噛ませる。**

```ts
// 全角/半角カッコ・各種括弧の中身、☆や補足記号を除去してから読み上げる
const stripForSpeech = (text: string) =>
  (text || '')
    .replace(/[（(［\[【「『][^）)］\]】」』]*[）)］\]】」』]/g, '') // 括弧と中身
    .replace(/[☆★※→]/g, '')
    .replace(/\s+/g, ' ')
    .trim();
```
※ ただし「カッコの中だけが読みたい本体」のカードが無いか、投入データ(`content/*.json`)を一度確認してから正規表現を確定すること。

### 課題2: 自動読み上げ音声がカクつく／きれいに聞こえない
ブラウザ標準TTSは**端末・音声エンジン依存**で、特に以下で品質が落ちる:
- `getVoices()` が非同期ロードのため、ロード完了前に `speak()` するとフォールバック音声でガサつく。
- 既定音声が日本語に最適でない／`rate` を下げすぎるとブツ切れに聞こえる。
- `cancel()` 直後に `speak()` すると頭が欠ける。

→ **対策（順に効く）:**
1. **日本語音声を明示選択**：`getVoices()` から `lang.startsWith('ja')`（Macなら 'Kyoko' 等）を選ぶ。`voiceschanged` イベントでロード完了を待ってからキャッシュ。
2. `rate` は 0.9→**1.0** で試す（エンジンによっては等速のほうが滑らか）。
3. `cancel()` と `speak()` の間に最小ディレイ、または再生中フラグで連打を抑止。
4. **それでも品質が不十分なら「事前生成mp3」方式に切り替える**（下記「将来の品質担保案」）。Web Speech の品質はAndroidで特に低いため、全端末同品質を求めるならこちらが本命。

```ts
// 日本語音声を一度だけ解決してキャッシュ
let jaVoice: SpeechSynthesisVoice | null = null;
const pickJaVoice = () => {
  const vs = window.speechSynthesis.getVoices();
  jaVoice = vs.find(v => /ja|Kyoko|O-ren|Otoya/i.test(v.lang + v.name)) || null;
};
if (typeof window !== 'undefined' && 'speechSynthesis' in window) {
  pickJaVoice();
  window.speechSynthesis.onvoiceschanged = pickJaVoice;
}

const speak = (text: string) => {
  if (!('speechSynthesis' in window)) return;
  const u = new SpeechSynthesisUtterance(stripForSpeech(text));
  u.lang = 'ja-JP';
  if (jaVoice) u.voice = jaVoice;
  u.rate = 1.0;
  window.speechSynthesis.cancel();
  window.speechSynthesis.speak(u);
};
```

### 3. B（発音判定）のゆるめ照合ロジック
recognizer は ja-JP で**漢字混じり**を返すことがあるため、`front`（漢字あり）と `reading`（ローマ字）の**両方**に照合し、どちらか通ればOK。さらに「漢字front × かな発話」のズレを吸収するため、発話かなを**ローマ字化**して `reading` と突き合わせる。

```ts
const kataToHira = (s: string) =>
  s.replace(/[ァ-ヶ]/g, c => String.fromCharCode(c.charCodeAt(0) - 0x60));
const normalize = (s: string) =>
  kataToHira((s || '').normalize('NFKC'))
    .replace(/[\s　、。・,.!?！？「」『』（）()ー－〜~]/g, '').toLowerCase();
const similarity = (a: string, b: string) => { /* レーベンシュタイン距離→0〜1の類似度 */
  if (!a || !b) return 0; const m = a.length, n = b.length;
  const d = Array.from({ length: m + 1 }, () => new Array(n + 1).fill(0));
  for (let i = 0; i <= m; i++) d[i][0] = i; for (let j = 0; j <= n; j++) d[0][j] = j;
  for (let i = 1; i <= m; i++) for (let j = 1; j <= n; j++)
    d[i][j] = Math.min(d[i-1][j]+1, d[i][j-1]+1, d[i-1][j-1] + (a[i-1] === b[j-1] ? 0 : 1));
  return 1 - d[m][n] / Math.max(m, n);
};
// かな→ローマ字（ROMA1/ROMA2 テーブルは前回実装の五十音＋拗音＋っ対応版を再利用）
// const kanaToRomaji = (s) => ... ;

const isCloseEnough = (said: string, expected: string) => {
  if (!said || !expected) return false;
  const a = normalize(said), b = normalize(expected);
  if (a === b || a.includes(b) || b.includes(a) || similarity(a, b) >= 0.5) return true;
  const ar = kanaToRomaji(said), br = expected.replace(/[^a-z]/gi, '').toLowerCase();
  return !!(ar && br && (ar === br || ar.includes(br) || br.includes(ar) || similarity(ar, br) >= 0.5));
};
const judgePronunciation = (said: string, front: string, reading: string | null) =>
  isCloseEnough(said, front) || (!!reading && isCloseEnough(said, reading));
```

検証OKだった判定例（前回）: みず→水・あした→明日・がっこう→学校・カタカナ語・部分一致フレーズ＝○、別語＝×。

```ts
const listen = (target: { front: string; reading: string | null }) => {
  const SR = (window as any).SpeechRecognition || (window as any).webkitSpeechRecognition;
  if (!SR) return;
  const rec = new SR();
  rec.lang = 'ja-JP'; rec.interimResults = false; rec.maxAlternatives = 1;
  rec.onresult = (e: any) => {
    const said = e.results?.[0]?.[0]?.transcript ?? '';
    setResult({ ok: judgePronunciation(said, target.front, target.reading), said });
  };
  rec.onend = () => setListening(false); rec.onerror = () => setListening(false);
  setListening(true); rec.start();
};
```

## 将来の品質担保案（課題2が消えない場合の本命）

**事前生成mp3方式**：無料TTS（例: ローカルで Edge/VOICEVOX 等、または一度だけブラウザTTSを録音）で各カードの音声を mp3 生成 → Supabase Storage か `public/` に置き、`flashcards` に `audio_url` 列を足して `<audio>` で再生。
- 長所：全端末で同じ高音質・カクつかない。括弧問題も生成時に除去すれば解決。
- 短所：1,629枚分の生成・保存・管理コスト（ただし費用は0円で可能）。
- まずBが必要なデッキ（漢字/フレーズ）だけ生成、ひらがな等は不要、と段階導入できる。
