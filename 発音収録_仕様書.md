## 1. 概要・目的

日本語学習アプリ「Learning Support」のフラッシュカード機能に、**ネイティブによる単語・フレーズの手本音声**を追加する。学習者がカードをタップすると、当該語句の音声が再生される。本仕様書は、その**音声データ収録**を外部に依頼するためのもの。

- 利用者：日本語を学ぶ外国人（初級中心）
- 用途：正しい発音・アクセントの「お手本」を耳で確認する
- 収録言語：**日本語**

## 4. 音声仕様（ファイル形式）

| 項目 | 指定 |
|---|---|
| ファイル形式 | **MP3** |
| チャンネル | **モノラル** |
| ビットレート | **96 kbps**（128 kbpsまで可） |
| サンプリングレート | 44.1 kHz |
| 音量 | 全ファイルで**均一**になるようノーマライズ（目安 −3〜−1 dBFS ピーク、ラウドネス感を統一） |
| 無音 | 語句の**前後の無音は極力カット**（先頭 約0.1秒・末尾 約0.2〜0.3秒程度に整える） |
| ノイズ | 環境ノイズ・リップノイズ・クリッピング（音割れ）なきこと |
| 想定尺 | 1ファイルあたり概ね 0.5〜4秒 |

## 5. ファイル命名規則

CSVの **「ファイル名」列のとおり**に付与する。形式は次のとおり（発注者側でアプリと突き合わせるため厳守）。

```
<デッキスラッグ>_<3桁連番>.mp3
例）genki1-vocab-1_001.mp3 ／ n5-kanji_080.mp3 ／ youbi_013.mp3
```

- 連番はデッキ内番号（ゼロ埋め3桁）。
- **半角英数字・アンダースコアのみ**。全角・スペース・日本語をファイル名に含めない。
- 欠番・重複を作らない（CSVの行数＝ファイル数＝1,482）。

## 6. 読み上げガイドライン

1. **話者**：日本語ネイティブ。**全1,482ファイルを同一話者・同一マイク環境**で収録（声質・音量・距離を通して統一）。
2. **アクセント**：**標準語（東京式）アクセント**。
3. **速度**：**ゆっくり・はっきり**。初級学習者がそのまま真似できる明瞭さを最優先（早口にしない）。
4. **トーン**：平坦な棒読みではなく、自然で聞き取りやすい発話。語尾を不自然に上げない。
5. **1ファイル＝1語句**を、原則**1回だけ**読む（繰り返さない）。
6. **読む文字列は CSV「収録テキスト」列**。「表示語句(front)」ではない点に注意。
   - 例：表示「じゃあ、また (ね)」→ 収録テキスト「じゃあ、また」を読む（**括弧内は読まない**）。
   - 「読み(参考)」「意味(参考)」は発音判断の補助情報であり、読み上げ対象ではない。
7. **数字・日付・曜日**：CSVの収録テキストに従う（例：日付は「ついたち」「ふつか」等、語句どおりに読む）。
8. **判断に迷う語句**（複数の読み方・アクセントがありうる等）は、勝手に決めず**一覧にまとめて発注者へ確認**する。

## 7. 品質基準・NG例

**合格基準**

- 指定形式（MP3/モノラル/96kbps）で再生でき、全端末（スマホ含む）で問題なく聞ける。
- 1,482ファイルが揃い、CSVのファイル名と完全一致、欠番・重複なし。
- 語句が明瞭に聞き取れ、前後の余分な無音・ノイズがない。
- 音量が全体で均一。

**NG例（再収録・修正対象）**

- 音割れ／ホワイトノイズ／エアコン等の環境音が目立つ。
- 言い間違い・別語・余計な一言（「えーと」等）が入っている。
- 括弧内まで読んでいる／表示語句を読んでしまっている。
- ファイル名の誤り・連番ずれ・欠番。
- ファイル間で音量差が大きい。

- **フォルダ構成**：デッキスラッグごとにサブフォルダを切り、その中にMP3を格納。

```

├─ genki1-vocab-2/   (389ファイル)
├─ n4-kanji-phrases/ (229ファイル)
├─ phrases-100/      (122ファイル)
├─ katakana-words/   (100ファイル)
├─ n5-kanji/         (80ファイル)
├─ suuji/            (55ファイル)
├─ n5-kanji-phrases/ (50ファイル)
├─ hizuke/           (31ファイル)
└─ youbi/            (13ファイル)
```

- 全体を **ZIP** にまとめて納品（オンラインストレージ等のダウンロードリンク可）。
- 納品時に、ファイル総数（1,482）と欠番がないことを確認のうえ報告。
